# Calendar 3D Paper Realistic Variant

Автономный WebGL-прототип отрывного календаря с реалистичной бумажной страницей, неподвижной проволочной спиралью и настраиваемой физикой перелистывания.

Открывать напрямую:

```text
prototypes/calendar-3d-paper-realistic/index.html
```

## Что внутри

- WebGL-сцена с листами календаря и depth-test.
- Отдельные canvas-слои для аппаратной части календаря.
- Проволочная спираль, стержень, шайбы и тени вокруг отверстий.
- Покадровый слайдер `Кадр` для просмотра перелистывания по фазам.
- Панель `Параметры` для подбора поведения листа без правки кода.
- Сохранение состояния настроек в `localStorage`.

## Панель параметров

Доступные ручки:

- `Смена слоя` — фаза, после которой переворачиваемый лист логически уходит за новый лист.
- `Активный z` — визуальная глубина активного листа.
- `Старт раскрытия` / `Финиш раскрытия` — диапазон, в котором следующий лист начинает считаться текущим.
- `Задержка диагонали` — насколько перелистывание идет диагональной волной.
- `Длина волны` — растянутость волны переворота.
- `Боковой изгиб` — мягкий поперечный изгиб бумаги.
- `Подъем края` — дополнительный подъем боковых краев.
- `Микро-дрожь` — слабое движение бумаги во время перелистывания.
- `Писать depth после слоя` — проверка поведения depth-buffer после логического ухода листа назад.

Кнопки:

- `Сохранить` — фиксирует текущие параметры как удачную точку.
- `Сброс` — возвращает параметры к сохраненной точке.
- `Скрыть` — сворачивает панель.

На слайдерах золотая вертикальная метка показывает сохраненное значение. Красная дорожка показывает текущее значение.

## Текущая сохраненная точка

Стартовые значения зафиксированы как рабочий вариант:

```js
{
  layerSwitch: 0.56,
  activeZ: -0.01,
  revealStart: 0,
  revealEnd: 0.85,
  diagonalDelay: 0.015,
  flipSpan: 0.66,
  sideCurl: 0.015,
  edgeLift: 0,
  wobble: 0.0035,
  behindWritesDepth: false
}
```

При изменении ручек текущее состояние автоматически сохраняется в браузере. После перезагрузки прототип открывается с последними текущими настройками. Если нужно перенести найденные значения в код как дефолт, нажать `Сохранить`, затем скопировать значения из панели/кода.

Текущий основной проект не изменяется: это изолированный стенд в `prototypes/`.
