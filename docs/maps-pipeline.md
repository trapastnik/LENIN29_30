# Pipeline производства карт

## Концепция

Источник технологии — [map_v6.ipynb](https://github.com/trapastnik/LENIN29_30) (Jupyter в `~/Downloads/`). Ту же механику переиспользуем для всех 15 карт, интегрируя в проект.

**Вход:**
1. Оригинал карты (jpg/png) — исторический скан или отрисовка.
2. Слой линий от Nanabanana (PNG, цветные линии на прозрачном/чёрном фоне).

**Выход:**
`public/content/maps/<id>/`:
- `layers.svg` — многослойный SVG, один `<g id>` на цветовую группу линий
- `background.jpg` — растровый оригинал (отдельно, НЕ вшит в SVG)
- `thumb.jpg` — превью 320×180
- `map.json` — метаданные, список слоёв

## Шаги (повторяют map_v6.ipynb)

1. **HSV-сегментация**. Переводим Nanabanana-PNG в HSV, для каждого слоя — диапазон hue/saturation/value:
   - borders_dark (чёрные линии): `v<100 & s<80 & alpha>30`
   - arrows_red (КЦ): `(h<10|h>170) & s>60 & v>50`
   - arrows_pink: `140<h<172 & s>30`
   - front_blue: `95<h<135 & s>50 & v>60`
   - front_green: `35<h<85 & s>40`
   - rivers_cyan: `75<h<100 & s>15 & v>120`

2. **skimage cleanup**:
   - `remove_small_objects(min_size=30)` — убрать шум
   - `binary_closing` — закрыть мелкие разрывы в линиях
   - `skeletonize` — для стрелок (опционально)

3. **Potrace** каждого бинарного слоя → SVG path'ы.

4. **Сборка `layers.svg`**:
   - единый `<svg viewBox="0 0 W H">`
   - каждый слой в `<g id="<layer>" fill="<color>" transform="translate(0,H) scale(0.1,-0.1)" visibility="visible">`
   - путь остаётся в изначальных координатах potrace (потом `<map-unit>` применит свои transform)

5. **Вычистка base64**: в отличие от `map_v6/viewer.html`, растровый фон НЕ вшивается в SVG. Сохраняется отдельно `background.jpg` (sharp, quality 82, mozjpeg) и подтягивается компонентом как отдельный `<image>`.

6. **Генерация `map.json`** — meta из YAML + список слоёв.

7. **Thumbnail**: 320×180 JPG.

## Будущее: `scripts/build_maps.py`

Запланирован как batch CLI:
```bash
python scripts/build_maps.py --map <id>
python scripts/build_maps.py --all
```

Вход: `input/maps_source/<id>/` с `source_raster.jpg` + `nanabanana.png` + `meta.yaml`. Выход: `public/content/maps/<id>/` со всем комплектом.

Сейчас есть только [trace_venn_blobs.py](../scripts/trace_venn_blobs.py) — связанный трейсер, но он для Венн-диаграммы партий (HSV-маски + cv2 findContours → Catmull-Rom), не для карт. Архитектура та же, код надо адаптировать.

## Трудозатраты

Каждая карта — ~30-90 минут ручной работы (HSV-пороги подбираются по цвету линий). Batch-скрипт должен автоматизировать большую часть, оставив только рутину ретуши слоёв.

Итого 14 карт × ~60 мин ≈ 14 часов. Производство — Phase E по плану (M4).
